사진 적용됨

기본 상태는 이름 첫 글자(data-mypage-avatar-initial)를 원형 배경에 표시하고, 사진이 있으면 <img data-mypage-avatar-img>로 교체합니다. 전환은 JS setPhoto(applied)가 담당 — img.style.display과 이니셜 display를 토글.

0진입점 (Depth 0)
화면 마이페이지 / 기본정보 수정 진입

두 화면 모두 동일한 아바타 마크업([data-mypage-avatar-wrap])과 동일한 스크립트(initMypageAvatarCrop)를 공유합니다. 카메라 버튼([data-mypage-avatar-edit])이 유일한 시작 트리거.

📍 초기 상태 — 로드 시 사진 없음(hasPhoto = false). 이니셜 “준” 표시.
1카메라 버튼 탭 → 사진 유무 분기 (Depth 1)
액션 카메라 아이콘 탭

editBtn.click → 현재 hasPhoto 값으로 즉시 두 갈래.

IF · 사진 없음

파일 선택기 바로 열기

fileInput.click() 호출 → 메뉴 건너뜀. → Depth 3으로 점프.

IF · 사진 있음

액션 메뉴 오버레이 열기

toggleMenu() → Depth 2 (변경 / 기본으로 / 취소).

2액션 메뉴 (사진 있을 때만) (Depth 2)
오버레이 프로필 사진 메뉴 [data-mypage-avatar-menu]

3개 선택지. 바깥 탭·Esc로도 닫힘(변경 없음).

선택 A 사진 변경

closeMenu() 후 fileInput.click() → Depth 3.

선택 B · 종료 기본 사진으로

setPhoto(false) + img.src = "" → 이니셜 “준” 복귀. 플로우 종료 (사진 제거됨).

선택 C · 종료 취소

closeMenu() — 아무 변경 없이 메뉴만 닫힘.

3파일 선택 → 이미지 로드 (Depth 3)
액션 OS 파일 선택기에서 이미지 선택

<input type="file" accept="image/*">. 선택 시 openModal(file) 호출, 이후 fileInput.value = ""로 같은 파일 재선택 허용.

↩︎ 파일 선택 취소 — 선택기를 닫으면 change 미발생, 아무 일도 없음(이전 상태 유지).
처리 FileReader → Image 로드 → 크롭 모달 오픈

readAsDataURL → new Image() onload 시 초기 배율 계산:

baseScale = max(280/w, 280/h) (캔버스 꽉 채움), scale = 1, offset 0, 줌 슬라이더 1 리셋 후 오버레이 표시.

4원형 크롭 편집 (Depth 4)
모달 프로필 사진 편집 [data-mypage-crop-overlay]

280×280 캔버스, 원형 클립 미리보기. 권장 256×256px↑ 정사각형 안내.

조작 1 드래그로 위치 이동 (팬)

마우스·터치 드래그로 offsetX/Y 변경. clampOffset()이 이미지가 원 밖으로 빠지지 않게 경계 제한.

조작 2 슬라이더로 확대/축소 (줌)

[data-mypage-crop-zoom] 범위 1 ~ 3, step 0.01. 값 변경 시 scale 갱신 → clamp → 재렌더.

취소 · 종료

취소 / 바깥 탭 / Esc

closeModal() — source = null, 사진 미적용. 이전 상태 그대로.

적용

적용 → 렌더 & 저장

256×256 출력 캔버스에 원형 클립 후 toDataURL("image/png") → img.src 세팅.

5완료 (Depth 5)
완료 · 변경됨 프로필 사진 적용 완료

setPhoto(true) → 이니셜 숨김·사진 표시, closeModal(). 이후 카메라 탭 시 이제 Depth 2 메뉴가 열림(사진 보유 상태).

🔁 루프 — 사진 보유 후에는 “사진 변경”으로 Depth 3~4 재진입, “기본 사진으로”로 초기 상태 복귀 가능.
🧭

구현 참고

이 플로우 전체는 initMypageAvatarCrop() 하나에 담겨 있고 mypage.html·edit.html이 동일 마크업으로 재사용합니다. 아이콘은 lucide(CDN), 크롭은 순수 Canvas 2D로 외부 라이브러리 없이 구현됨. 현재는 클라이언트 미리보기 전용이며 서버 업로드 단계는 미포함.
